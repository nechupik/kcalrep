# Промпты для Windsurf - исправление проблем в kcal приложении

## Проблема 1: Модальное окно не раскрывается полностью

**Проблема:** Модальное окно добавления еды (AddFoodModal) не раскрывается полностью после изменений анимации в профиле. Высота модального окна ограничена и не показывает всё содержимое.

**Корневая причина:** В AddFoodModal.tsx используется динамический расчет высоты через `contentHeight` и `contentRef`, но анимации из Profile.tsx могут конфликтовать с CSS переходами.

**Файл для исправления:** `src/components/AddFoodModal.tsx`

**Что нужно сделать:**
1. Убедиться, что `contentHeight` правильно рассчитывается при изменении вкладок и контента
2. Проверить, что CSS переходы не конфликтуют с динамической высотой
3. Возможно нужно добавить `min-height` для модального окна
4. Проверить, что `useEffect` для расчета высоты срабатывает после рендера контента

**Конкретные строки для проверки:**
- Строки 38-40: состояние для высоты контента
- Строки 53-58: useEffect для расчета высоты
- Строки 235-241: контейнер с динамической высотой
- Строки 183-197: стили модального окна

## Проблема 2: Кнопка "пересчитать норму" не работает

**Проблема:** В профиле кнопка "Пересчитать норму" не сохраняет новые данные и не обновляет норму КБЖУ.

**Корневая причина:** В Profile.tsx функция `handleSaveNorm` не передает параметры (gender, height, age, goal) в функцию `saveNorm`, поэтому Firestore не сохраняет полные данные нормы.

**Файл для исправления:** `src/pages/Profile.tsx`

**Что нужно сделать:**
1. Добавить состояние для хранения параметров расчета нормы (gender, height, age, goal)
2. В функции `handleCalculate` сохранять не только результат, но и входные параметры
3. В функции `handleSaveNorm` передавать сохраненные параметры в `saveNorm`

**Конкретные строки для исправления:**
- Строки 39-42: добавить состояние для параметров расчета
- Строки 135-138: обновить `handleCalculate` для сохранения параметров
- Строки 140-147: исправить `handleSaveNorm` для передачи параметров

## Проблема 3: Отсутствует кнопка отмены при удалении

**Проблема:** При удалении блюд/продуктов из дневника нет подтверждения, что может привести к случайному удалению.

**Корневая причина:** Функции `handleRemoveEntry` в Index.tsx и Diary.tsx выполняют удаление сразу без диалога подтверждения.

**Файлы для исправления:** 
- `src/pages/Index.tsx`
- `src/components/Diary.tsx`

**Что нужно сделать:**
1. Импортировать AlertDialog компонент из UI
2. Создать состояние для показа диалога подтверждения
3. Добавить AlertDialog с кнопками "Отмена" и "Удалить"
4. Обернуть удаление в диалог подтверждения

**Конкретные места для добавления:**
- Index.tsx: строки 113-118 (функция handleRemoveEntry)
- Index.tsx: строки 479-483 (кнопка удаления)
- Diary.tsx: аналогичные места для удаления
